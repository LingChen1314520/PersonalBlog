# 构建 Nova：在代码中寻找极简主义的平衡点

为什么在 2025 年，我还要从零开始构建一个个人博客系统？

## 1. 核心设计初衷
在信息爆炸的时代，我渴望一个**专注、干净、且完全由我控制**的数字空间。Nova 的设计遵循“内容优先”的原则，去除了所有干扰阅读的侧边弹窗和繁琐装饰。

## 2. 技术栈选型
为了保证极致的性能和开发体验，我选择了：
- **React 19**: 利用最新的 `use` 钩子和更简洁的异步渲染逻辑。
- **Vite**: 提供秒级的开发热重载，让创意不间断。
- **Tailwind CSS**: 通过原子化类名快速实现磨砂玻璃（Glassmorphism）视觉风格。
- **Marked.js**: 用于将静态 Markdown 资产实时转化为精美的网页内容。

## 3. 极简并不意味着简单
虽然界面极简，但 Nova 内置了完整的**身份验证管理后台**。这意味着我可以在不打开代码编辑器的情况下，通过 UI 实时调整项目权重、更新链接或切换内容状态。

## 4. 未来的进化
接下来，我计划引入 **Gemini API** 来为长文章自动生成摘要，并加入基于语义搜索的全局搜索功能。
